## 2. S1-2 算法说明

### 2.1 上一版（C1）的问题

上一版 `match_sentence` 使用**锚点聚类法**：
1. 在转写全文中找所有 n-gram 锚点
2. 按"首锚点起始位置差 > sent_len × 0.8"切分 cluster
3. 每个 cluster 做局部滑动窗口打分

**已确认的 Bug（句6 漏匹配）：**
- 句6（47字）在转写中对应约 50 字符区域（含大量即兴插词）
- 锚点跨度 50 chars > 聚类阈值 47×0.8=38 chars，被切成两个 cluster
- Cluster 0 窗口被截止到 Cluster 1 起始位置，失去右侧关键 n-gram
- 两个 half-cluster 各自 F1 仅 0.18，低于阈值 0.25 → 漏匹配

### 2.2 新算法（Dev C2）：全局峰值检测

**核心思路**：不再聚类，改为全文评分 + 局部极大值检测。

```
对每个转写起始位置 t_s：
  用滚动 n-gram 计数器，找 [WIN_MIN, WIN_MAX] 长度范围内最高 F1
  → score_array[t_s], end_array[t_s]

在 score_array 中找所有局部极大值（NMS 半径 = max(ng, q_len×0.8)）
  → 自动处理单次匹配和多次重复（NG 重拍）

对每个保留的极大值峰：
  映射字符位置 → 词索引 → 时间戳
  加权置信度 = F1 × 转写词平均置信度因子
```

**滚动计数器优化**：每步只增/减 1 个 n-gram，比重建 Counter 快约 3-5x。

**NG 重拍分离保证**：NMS 半径 = max(4, q_len × 0.8)。
- 句9"这一次有什么不一样呢"（q_len=10）→ NMS 半径 = 8
- 两次出现间距 = 10 字符 > 8 → 两个峰值均保留，重复检测成功

### 2.3 为什么句6 现在能匹配

新算法从 t_s=130 开始评分，滚动扩展窗口从 30 到 95 chars，
在 t_s=130, sub_len=50 时达到最高 F1=0.264 > 阈值 0.25，
该位置恰好对应"当时的理由其实非常合理因为要所谓的牛出乳…"，语义正确。

## 5. 可调参数清单

所有参数集中在 `src/align.py` 顶部 `CFG` 字典，无需修改代码逻辑。

| 参数 | 默认值 | 作用 | 调节建议 |
|---|---|---|---|
| `NGRAM_SIZE` | 4 | n-gram 长度，影响字序敏感度 | 中文短词多时降为 3；长词文本可升为 5 |
| `WIN_MIN_RATIO` | 0.65 | 候选窗口最小长度系数（× q_len） | 调小 → 容忍更短的匹配（但可能误匹配） |
| `WIN_MAX_RATIO` | 2.2 | 候选窗口最大长度系数（× q_len） | 调大 → 容忍更多即兴插词；调小 → 精准型素材 |
| `NMS_RADIUS_RATIO` | 0.8 | NMS 半径系数（× q_len） | **关键**：必须 < 两次 NG 重拍的间距比。调大会合并重复，调小会产生碎片匹配 |
| `MIN_MATCH_CONFIDENCE` | 0.25 | 最低 F1 置信度阈值 | 调低 → 匹配更多但精度下降；调高 → 更精准但可能漏匹配 |
| `ANCHOR_CONF_THRESHOLD` | 0.5 | 转写词置信度打折阈值 | 低置信度转写可降为 0.3；高质量音频可升为 0.7 |
| `ADLIB_MIN_DURATION` | 0.5s | 脱稿区间最短时长 | 调大 → 过滤短暂脱稿（如"嗯"）；调小 → 记录更多细节 |
| `MIN_MATCH_CONFIDENCE` × `ANCHOR_CONF_THRESHOLD` | — | 组合调节 | 即兴偏稿大时两者均调低；精准朗读时均调高 |
| `KEEP_LAST_REPEAT` | True | 重复区间保留最后一次 | 改为 False → 保留第一次；FR-3 规定默认保留最后一次 |
| `SCRIPT_START_MARKER` | "视频号版（终稿）" | 定稿文件中提取节名 | 飞书文档标题不同时修改此值 |
