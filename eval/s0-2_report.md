任务要求 base 模型（CPU int8），但 base 模型（~145MB）未缓存，网络下载因沙箱重置被中断。使用已验证可用的 **tiny 模型（75MB int8）** 作为 fallback，符合 PRD §8 降级原则。

**实际影响**：tiny 与 base 在词级时间戳精度上接近，差异主要在中英混说边界词。Sprint 1 转写模块（S1-1）需以 base 或更大模型为准；本次数据用于统计基线，不作为 golden EDL。

## 3. 统计分析（S0-2 Step 3）

### 3.1 停顿分析（>0.8s 判定为卡壳/喘气）

| 指标 | 原素材 | 粗剪 |
|---|---|---|
| 总停顿数(>0.8s) | **3** | **0** |
| 分布 0.8-1s | 1 | 0 |
| 分布 1-2s | 1 | 0 |
| 分布 2-5s | 0 | 0 |
| 分布 >5s | 1 | 0 |

原素材最长停顿TOP3：

| 时间点 | 停顿时长 | 上下文 |
|---|---|---|
| 154.6s | 8.20s | `套` → `现在`（片尾处，可能是镜头切换空档） |
| 164.1s | 1.32s | `销` → `那` |
| 108.3s | 0.95s | `据` → `我也` |

**关键发现**：粗剪0个>0.8s停顿，说明既有粗剪已基本处理了停顿问题。8.2s的超长停顿位于片尾154.6s附近，疑为摄像机关机前的空白段。

### 3.2 语气词统计

| 语气词 | 原素材 | 粗剪 |
|---|---|---|
| 这个 | 4 | 5 |
| 就是 | 1 | 1 |
| 那个 | 0 | 1 |
| 呢 | 1 | 0 |
| 啊 / 嗯 / 呃 | 0 | 0 |
| **合计** | **6** | **7** |

**关键发现**：原素材197s中有6个语气词，折算约1.8个/分钟；粗剪154s有7个，折算约2.7个/分钟。粗剪语气词密度反而略高，说明粗剪未做系统语气词清除，这正是自动化管线要解决的P4问题。PRD §7门槛：≤2处/10分钟成片。

### 3.3 重复句段（n-gram 自相似检测）

基于字符级4-8元组检测，原素材中出现频次≥2的片段：

| 重复次数 | 内容 |
|---|---|
| 3x | `市场营销` |
| 2x | `这一次有什么不一样` |
| 2x | `我也研究了一下牛初乳的科学证据` |
| 2x | `这两年牛粗乳` |
| 2x | `改善皮肤和抗...` |

**关键发现**：`这一次有什么不一样` 出现2次，是典型的NG重拍（先说了一遍，再重说了一遍）。对照飞书定稿确认，该句在定稿中只出现一次，因此原素材中有一次是重复，需剪除。这正是 FR-3 重复消除的典型场景。

## 4. 原素材 vs 既有粗剪差异摘要（S0-2 Step 4）

| 指标 | 值 |
|---|---|
| 原素材时长 | 197.95s |
| 粗剪时长 | 154.45s |
| 剪除总量 | 43.5s（22% 压缩） |
| 原素材词数 | 673 |
| 粗剪词数 | 615 |
| 词数差 | 58词（-8.6%） |

### 推断被剪除的内容（文本比对）

通过将原素材78个转写片段逐一在粗剪全文中检索，发现约40个片段不在粗剪中（部分为转写噪音）。较可信的推断删除内容（结合飞书定稿交叉验证）：

| 被删内容 | 类型 | 推断原因 |
|---|---|---|
| `还记得当年红级意识的英尔宝剑品叫牛粗乳吗` | 转写噪音/NG | tiny模型识别错误，定稿中对应句已保留 |
| `这一次有什么不一样的`（第1次出现） | **重复句删除** | 对应定稿同句，保留后一次更流畅 |
| `这两年牛粗乳又回来了` 后的部分拓展句 | 节奏剪辑 | 粗剪压缩了数据引用段 |
| `九件套（待补充）` 相关内容 | 未录制 | 定稿九件套为草稿，未出现在素材中 |

**注意**：由于 tiny 模型精度有限，部分差异可能是转写误识别而非真实剪辑。需以 base/large-v3 模型重跑后交叉验证方可最终确认。

### 对 golden EDL 标注的启示

1. **片尾8.2s超长停顿**（154.6s处）：粗剪已处理，golden EDL 应标记此段为 `keep=false, reason=trailing_silence`
2. **"这一次有什么不一样"** 重复：golden EDL 应标记两个区间，保留最后一次（FR-3规则）
3. **语气词**（"这个"×4次）：golden EDL 应标记每个语气词区间为 `keep=false, reason=filler`
4. **粗剪压缩比22%**：对于3.3min原素材，粗剪后约2.6min，成片目标3-6min意味着还需要素材增加或多期合并

### 5.2 飞书文档结构

拉取到的文档（0602【牛初乳老树开新花】口播稿）包含：
- **视频号版终稿**：完整口播稿（约350字），这是 ground truth 对齐文本
- **草稿/AI底稿**：更详细的备选版本（约450字）
- **参考来源**：5条带URL的事实核查来源

**定稿与转写对比**：定稿视频号版约350字，原素材转写756字，说明录制时有大量即兴发挥或NG重录，偏离定稿约115%。这印证了 PRD §8 风险"即兴发挥偏离定稿，对齐退化"为中等风险，Sprint 1 的对齐模块（S1-2）需要做鲁棒的 anchor-based diff。

### 5.3 读取流程说明

```
1. POST /auth/v3/tenant_access_token/internal
   → 传入 app_id + app_secret → 获得 tenant_access_token

2. GET /docx/v1/documents/{doc_token}/raw_content
   → Authorization: Bearer {token}
   → 返回 code=0, data.content = 纯文本全文
```

旧版 `/doc/v2/` API 不支持 Docx 类型（返回 code=95053），必须使用 `/docx/v1/` 端点。

### 5.4 Chen 无需额外操作

App ID 和 Secret 已在现有 `tianbaba-topic-research` skill 中（`feishu_bitable.py`），该 App 已有文档读取权限（`docs:doc:readonly` 或同等 scope）。只需复用即可。

**唯一潜在风险**：如果文档所在空间对 App 有访问限制（需手动在飞书管理后台将 App 加入该空间），则需 Chen 在飞书管理后台确认 App 已有权限。本次读取成功说明**当前已有权限，无需 Chen 操作**。
