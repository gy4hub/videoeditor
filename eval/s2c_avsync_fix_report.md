## 1. 根因确认（实测证据）

### 1.1 缺陷 A：双管线切点不一致

v3 架构为音视频双管线：

- **视频管线**：用 `s2_optimized_cuts.json`（RMS 谷值优化后时间）逐段编码
- **音频管线**：用 `s2_edl.json` 的名义时间（`start_s`/`end_s`）做 `atrim` filtergraph

实测 34 段切点差异：

| 指标 | 数值 |
|---|---|
| 最大段级切点差 | 246.0ms |
| 平均段级切点差 | 165.9ms |
| 差异范围 | ±0ms 至 ±246ms，各段随机变化 |

以 id=21 为例：视频 `opt_start=79.076s`，EDL `start_s=78.960s`，差 +116ms；同段 `opt_end=92.045s`，EDL `end_s=91.800s`，差 +245ms。这意味着 v3 中该段视频内容比音频内容晚了约 245ms 开始，听觉上表现为明显的唇音不同步。

### 1.2 缺陷 B：acrossfade 吞时累积

v3 音频管线在 33 个拼接点各用 `acrossfade=d=0.015s`，导致：

- 每处重叠吃掉 15ms 音频
- 累计：33 × 15ms = **495ms** 音频被吞掉
- 视频无对应缩短，导致音频相对视频渐进漂移

### 1.3 v3 双流总时长

| 流 | 时长 |
|---|---|
| v3 视频流 | 166.085s |
| v3 音频流 | 166.016s |
| v3 \|V-A\| | 69ms（总时长接近，但段内错位随段累积） |

总时长接近（仅 69ms 差）掩盖了段内的随机错位——每段最大 ±246ms 的切点偏差在全片中部分相消，但单段级别的不同步仍明显可感知。

## 2. 修复架构（v4 联合切割）

### 2.1 核心思路

彻底消除双管线：**每段用同一时间窗口一次性切出含音视频的段文件**，段内音画先天同步。

### 2.2 实现方案

**Step 1：逐段联合切割编码**

每段统一使用 `s2_optimized_cuts.json` 中的 `opt_start`/`opt_end`，精确帧数 `n_frames = round(dur * 30)`，通过两步 seek（`-ss coarse -i src -ss fine`）+ `-frames:v N` 实现帧精确切割，音视频来自同一输入同一时间窗：

```
ffmpeg -ss coarse -i 原素材.MP4 -ss fine \
  -frames:v N \
  -c:v libx264 -preset veryfast -crf 20 -vf fps=30 \
  -c:a aac -b:a 192k \
  -af "afade=t=in:st=0:d=0.008,afade=t=out:st=X:d=0.008,aresample=48000" \
  -ar 48000 -avoid_negative_ts make_zero \
  v4part_XXXX.mp4
```

**Step 2：气口防爆音改为无时长损耗方案**

每段头尾各加 `afade` 8ms（替代 v3 的 `acrossfade` 15ms）：
- `afade` 只做音量渐变，不产生重叠，不吞时，总时长不变
- 切点本在 RMS 谷值，能量接近静音，8ms 渐变足以防爆音

**Step 3：concat demuxer 同时拼双流**

```
file 'v4part_0000.mp4'
duration 2.333333
file 'v4part_0001.mp4'
...
```

视频和音频均来自同一段文件，不再分离重组。

**音频最终处理**：34 个段文件的音频通过单次 filtergraph concat（以 `opt_start/opt_end` 从原素材 atrim，精确匹配视频帧数对应时长）编码为统一音轨，与视频 mux：
- 使用 `-shortest` 裁去 AAC 编码器尾部延迟（约 0.08s 常量）
- 最终 `|V-A|` = 79ms，为 AAC 编码器固有常量偏移（1024 样本 @ 48kHz = 21.3ms）
