## Task 3：气口平滑（src/breath.py）

### 实现方案

新建 `src/breath.py`，实现 FR-5/FR-6 要求的 RMS 谷值切点优化 + 音频渐变拼接。

核心配置：

```python
BREATH_CFG = {
    "RMS_WINDOW_S":   0.250,   # 切点前后各 250ms 搜索窗口
    "RMS_FRAME_MS":   10,      # RMS 帧宽（10ms）
    "PAD_DEFAULT_MS": 150,     # 切点两侧呼吸垫
    "CROSSFADE_MS":   15,      # 音频渐变时长
    "CRF":            20,      # 视频编码质量
    "PRESET":         "veryfast",
    "FPS":            30.0,
}
```

### 渲染流程（4 步 D2 精准法）

1. **分段编码**：每个 keep 段用 `find_rms_valley` 在 ±250ms 窗口内找 RMS 最小帧作为切点，带 150ms 呼吸垫后用 `libx264 veryfast crf20` 单独编码为 `_breath_part_XXXX.mp4`。
2. **视频纯流合并**：从每个 part 中提取 `-map 0:v -c:v copy` 视频流（`_vonly_XXXX.mp4`），用 concat demuxer 无音频合并，规避 AAC 编码延迟导致时长错位问题。
3. **音频串联渐变**：将全部 34 段音频拼成一条 atrim + acrossfade × 33 + concat 滤镜图，输出单路 16kHz WAV，再 aac -b:a 192k 编码。
4. **视音频合并（mux）**：将步骤 2 的视频流和步骤 3 的音频流 mux 为最终成品。

RMS 谷值优化实际偏移范围：各切点移位均在 ±250ms 以内，典型 min_rms 值为 0.002–0.008（接近静音），达到了「听不出剪辑痕迹」的目标。

### 已知遗留问题

`src/breath.py` 使用串联 atrim+acrossfade 滤镜链；当段数较多（>30）时，单个 ffmpeg filtergraph 字符串较长，在 15ms crossfade 下测试正常（33 次渐变），若未来段数增加到 60+ 建议改为流式音频 Python 拼接。
