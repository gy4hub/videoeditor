# S2c 音画同步返工验收（Inspector: Claude / 2026-06-12）

## 缺陷回顾

Chen 打回 v3 严重音画不同步。根因（dev 实测确认）：双管线架构下视频用谷值优化切点、音频 filtergraph 切点与之最大相差 246ms/段（平均 166ms），叠加 33 处 acrossfade 累计吞掉 495ms 音频。总时长几乎一致掩盖了段内错位——这也解释了为什么上次验收的时长检查没拦住它。

## 修复与验证

| 项 | 结果 |
|---|---|
| 新架构 | 联合切割：每段音视频同窗口一次切出（帧精确），段内先天同步；acrossfade→8ms afade（零时长损耗）；concat 双流同拼 |
| 音画同步硬检 | 5 段（含首/中/末）互相关 lag 全部恒定 +21.4ms（AAC 编码器固有 1024 样本延迟），相关峰值 0.97+；视频感知哈希全过。Inspector 独立复跑工具结论一致。恒定性本身排除漂移与随机错位 |
| v4 时长 | 166.5s，与理论 0.000s 偏差 |
| 流程改进 | 音画同步检查已集成进 qc_report.py §6，今后每版成片必检 |
