## 1. 背景与问题

现有 finecut（`skills/finecut/SKILL.md`）用 HyperFrames 的 **insert 整屏替换**模式：图形卡是纯黑底整屏 MP4，渲染时**切走真人画面**只留声音。2026-06-15 用真实素材跑通后，Chen 验收暴露三个核心问题：

1. **割裂**：纯黑整屏卡把"阳光户外真人"切成"黑屏数据卡"，与原片风格对比最强烈。
2. **一闪而过**：卡片仅 3–5s，入场动画占去大半，可读时间太短。
3. **结合差**：insert 本质是"切走再切回"，图形与口播是两层割裂的东西，不共存。

根因：范式选错了——用了 cutaway（切到整屏卡），而需要的是图形与真人画面**和谐共存**。

## 2. 目标与非目标

**目标**
- G1：图形与口播画面共存，添爸全程在画面里，过渡无割裂感。
- G2：skill 适配添爸**任意场景**素材（明/暗、室内外），不针对单条调参。
- G3：图形时长与口播内容贴合，不"一闪而过"。
- G4：可控——渲染前人能看到并修改"插入清单"，不浪费整片渲染。

**非目标（本次不做）**
- 不做选题/写稿/字幕烧录（字幕已有独立路径）。
- 不做 B-roll 寻源（属 S5）。
- 不改粗剪管线。

## 3. 核心设计决策（已评审通过）

### 3.1 合成模型 — A-roll 进 HyperFrames 视频轨（方案 A）
粗剪整条作为 `<video>` A-roll 轨（`data-track-index=0`）放进**单个** HyperFrames composition，所有叠加层按时间码挂在更高轨。**一次渲染**出"真人 + 叠加层"的合成片——无 ffmpeg 拼接、无拼接点跳色、天然无缝。

- 已验证：HyperFrames 原生支持 video 轨 + 叠加层（test-composition 即此结构），符合 PRD FR-11 原设计。
- 已验证：Chrome 渲染器**支持 CSS `backdrop-filter: blur`**（2026-06-15 真机预览渲染通过）。
- 代价：渲染整条 154s+ 比只渲卡片慢（Chrome 逐帧），3–6 分钟成片可接受。

### 3.2 叠加层视觉 — 磨砂玻璃 + 上方安全区
- **质感**：半透明磨砂玻璃面板，`backdrop-filter: blur(~28px) saturate(1.2)`，背景 `rgba(18,22,30,0.42)`，1px 浅描边，大圆角。透出背后真人虚化画面，明暗场景皆融。
- **位置**：**画面上部**（上方条 / 上方卡）。**底部永远留空**——留给烧录字幕 + 抖音/视频号平台 UI 遮罩。安全区约定：所有叠加层 `bottom` 不低于画面高度的 ~65%（即不进入下方 35%）。
- **强节点**：**全屏磨砂压屏**（真人虚化变暗 + 内容居中）。少用，留给最强的一两个节点。

### 3.3 时长与进出 — 跟着话走，自动费时
- 图形在添爸**开口讲这个点时淡入**，**讲完该句/该段才淡出**。
- 时长由 ASR 词级时间戳**自动推导**：插入点 = 该论点首词 start，结束 = 该论点末词 end（典型 6–15s）。
- 进出：淡入淡出 + 轻微位移（GSAP），不硬切。

### 3.4 模板集（替换现有 3 个纯黑整屏模板）
| 新模板 | 用途 | 位置 |
|---|---|---|
| `topbar` 磨砂条 | 关键词 / 小标题 / 单个数据 | 上方横条 |
| `stat` 磨砂卡 | 一个有冲击力的数字 | 上方角卡 |
| `chart` 磨砂图 | 多值对比 | 上方面板 |
| `fullscreen` 磨砂压屏 | 强节点章节感 | 全屏 |

现有 `chart-bar/chart-stat/text-highlight`（纯黑整屏）标记废弃；其 GSAP 动画与数据注入逻辑可复用，重做成磨砂上方版。所有模板：1080×1920，CJK 字体（见 §6 风险），`--variables` 注入内容、不写死样式（沿用 PRD FR-11 "LLM 只填内容不写样式"）。

### 3.5 选点规则与密度
- 多值对比 → `chart`；单个强数字 → `stat`；新名词 / 章节转场 → `topbar` 或 text；最强节点 → `fullscreen`。
- 数据来源：定稿 ground truth，**不捏造**；与口播口误冲突时以定稿为准（例：本片口播"300倍"、定稿"3000%/61万→1900万"，取定稿，用 chart 展示两个真实数值规避争议）。
- 密度：不背靠背；平均每 ~30s 不超过 1 个叠加层；`fullscreen` 全片 ≤ 2 个。

### 3.6 工作流 — 先出清单，确认再渲染
```
ASR 转写(词级时间戳)
  → Agent 读定稿 + 转写,产出人可读 finecut-spec
  → Chen 过一遍(改/删/调时间) → 确认
  → 渲染(A-roll + 叠加层,单 composition,一次渲染)
  → 出片 finecut.mp4
```
finecut-spec 是精剪的"EDL"（对应 PRD FR-11b video-spec）：人类可读、逐条可改、改后重渲染。

## 4. finecut-spec 格式

人可读 JSON（每条插入一项）：
```json
{
  "source": "粗剪_牛初乳老树开心花.mp4",
  "fps": 30,
  "width": 1080, "height": 1920,
  "inserts": [
    {
      "id": 1,
      "template": "chart",
      "placement": "upper",
      "at": "auto",
      "start_s": 54.6,
      "end_s": 60.9,
      "based_on": "美国这两年牛初乳产品从61万美元涨到了1900万美元",
      "script_source": "定稿_牛初乳.md L:美国市场段",
      "vars": {
        "eyebrow": "美国牛初乳市场 · 两年",
        "bars": [{"label":"原来","value":61,"unit":"万美元","color":"#888"},
                 {"label":"现在","value":1900,"unit":"万美元","color":"#4a9eff"}],
        "delta": "+3000%"
      },
      "note": ""
    }
  ]
}
```
约定：`at:"auto"` 时 start_s/end_s 由转写推导；Chen 手改后该项标 `edited_by:"human"`，重跑不覆盖。

## 5. 验证 / 验收

- **单模板**：四个新模板各 `hyperframes lint` 0 error；渲染样帧 Chen 确认磨砂质感、上方安全区、字体正常。
- **端到端**：以 `粗剪_牛初乳老树开心花.mp4` 跑全流程，产出 finecut-spec → 渲染成片；抽帧验证：①真人全程在 ②叠加层在上方、不压底部字幕 ③时长跟着口播 ④无割裂。
- **回归**：同 spec 两次渲染结果一致（无随机）。

## 6. 风险与开放点

| 风险 | 缓解 |
|---|---|
| CJK 字体在无苹方/黑体的机器回退（lint 警告） | 真机渲染用系统字体正常；若要可移植,补 woff2 嵌入模板（待定优先级） |
| 渲染整条 154s 慢 | 可接受（3–6 分钟片）；如太慢,退化为"只渲含叠加层的时间窗 + 拼接"（方案 A 变体,但有拼接点风险） |
| backdrop-filter 在某些素材上对比不足 | 面板加固定半透明底色兜底（已含 rgba 底），不纯靠 blur |
| "论点边界"自动推导不准 | spec 里 start_s/end_s 可人工微调；这正是确认闸的价值 |
