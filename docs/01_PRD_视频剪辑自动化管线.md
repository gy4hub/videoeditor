## 1. 背景与问题

现有工作流：飞书逐字稿（定稿）→ 真人出镜拍摄 → 粗剪 → 精剪。粗剪精剪目前由 agent 直接操作完成，存在五个核心痛点：

| # | 痛点 | 根因 | 本方案的解法 |
|---|---|---|---|
| P1 | 粗剪经常漏识别重复的话 | 依赖 LLM "看/听出"重复，注意力不可靠 | 用飞书定稿做 ground truth 文本对齐，重复 = 定稿同句匹配到多个素材区间，确定性算法，不会漏 |
| P2 | 气口切得生硬 | 在词边界硬切，无呼吸处理 | 切点外扩 100–200ms + 音频能量谷值定切点 + 10ms audio crossfade |
| P3 | hyperframe 精剪效果不稳定 | 调用参数随意、无重试与质检 | API 参数模板化 + 固定 preset + 输出质检 + 失败自动重试 |
| P4 | token 消耗过大 | GUI 截图驱动，每步消耗视觉 token | 转写/切割全本地（零 token），LLM 仅做文本决策，单期 ≤ 1 万 token |
| P5 | 出了问题难微调 | 无中间产物，只能整体重跑 | EDL 剪辑决策表作为中间产物，改一行重渲染 |

## 2. 目标与非目标

**目标**

- G1：输入原始素材 + 飞书定稿，一条命令产出可发布级粗剪成片 + EDL + 质检报告。
- G2：精剪半自动：hyperframe 处理、字幕、图表/动效、B-roll 装配，人在关键点确认。
- G3：单期粗剪 LLM 消耗 ≤ 1 万 token；同样输入产出可复现。
- G4：任意一刀剪错，5 分钟内通过改 EDL 修正并重渲染。

**非目标（本期不做）**

- 不做选题、写稿（已有 tianbaba 系列 skill 覆盖）。
- 不做发布上传。
- 不替代人工创意判断：B-roll 选材、图表内容由 agent 提案、人确认。

## 3. 总体架构

```
原始素材(mp4) + 飞书定稿逐字稿
        │
[粗剪 skill: rough-cut]
        ├─ ① 转写层：faster-whisper 本地词级时间戳（零 token）
        ├─ ② 对齐层：转写 ↔ 定稿文本对齐（anchor-based diff）
        ├─ ③ 决策层：规则引擎（停顿/语气词/空镜）+ LLM 文本决策（重复取舍/口误）
        ├─ ④ EDL：保留区间清单（人类可读、可改）
        └─ ⑤ 渲染层：ffmpeg 切割拼接 + 气口平滑 + 高清化滤镜
        │
   粗剪成片 + EDL + 质检报告
        │
[精剪 skills]
        ├─ fine-cut-hyperframe：API 调用模板 + 重试 + 输出质检
        ├─ subtitle-burn：中英字幕（规格见 FR-10）
        ├─ broll-graphics：图表/动效/B-roll 提案清单 → 人确认 → 装配
        └─ 风格对标：「小Lin说」呈现效果 checklist
        │
   成片（待 Chen 终审）
```

## 4. 功能需求

### 4.1 粗剪（skill: `rough-cut`）

**FR-1 转写**：faster-whisper（large-v3 或同级）本地转写，输出词级时间戳 JSON（词、start、end、confidence）。中英混说需正确分词。

**FR-2 稿件对齐**：从飞书拉取定稿（或接受本地文本输入），与转写做对齐。输出：每个定稿句子 → 素材中 0..n 个候选区间。

**FR-3 重复消除（2026-06-11 修订，S1 缺陷打回后强化）**：三道防线，缺一不可：(a) 定稿对齐重复——同一定稿句匹配多区间，默认保留最后一次；(b) 转写自重复——不依赖定稿，在转写流内检测时间邻近的 n-gram 自相似区间（捕获即兴句的 NG 重拍、结巴重启）；(c) LLM 语义复核——对全部疑似重复、全部脱稿区间逐段判断保留/删除并给理由（decided_by=llm）。删除区间写入质检报告。

**FR-0 交付形态（2026-06-11 Chen 修订）**：最终交付不是独立工具，而是打包成 skill（SKILL.md + scripts/），由 AI 执行。脚本负责确定性环节（转写/对齐/切割/渲染），AI 执行 skill 时自身充当语义决策层与质检员，并支持对话式微调（"把第 17 刀往后挪 0.3 秒"）。

**FR-4 语气词/口误剔除**：白名单（啊、嗯、这个、呃、那个、就是说…可配置）+ 时长阈值规则剔除；结巴/NG 重拍由对齐层捕获；歧义交 LLM 文本判断。

**FR-5 停顿处理**：词间隔 > 0.8s（可配置）判定为卡壳/喘气，剪除；保留首尾各 100–200ms 呼吸垫。首尾空镜（第一个词前 / 最后一个词后）自动剪除。

**FR-6 气口平滑**：切点定位到音频 RMS 能量谷值；拼接处 10ms audio crossfade；可选 2–4 帧视频交叉淡化。

**FR-7 EDL 输出**：格式见 §5。渲染严格按 EDL 执行，EDL 可手改后重渲染（`rough-cut render --edl xxx.json`）。

**FR-8 高清化滤镜**：ffmpeg unsharp + hqdn3d 基础档；可选 Real-ESRGAN 超分档（慢，按需开）。

**FR-9 质检报告**：每次运行输出：删除片段清单（时间码+文本+原因）、残留风险项（低置信度判断）、时长压缩比、token 消耗。

### 4.2 精剪

**FR-10 字幕（skill: `subtitle-burn`，基于现有 jianying-subtitle-style skill 改配）**：中英双语字幕；金陵体；中文 12 号、英文 8 号；白色；阴影不透明度 60%。英文译文由 LLM 从定稿生成（定稿是 ground truth，不用转写稿）。

**FR-11 HyperFrames 装配渲染（skill: `fine-cut-hyperframes`）**：HyperFrames（heygen-com/hyperframes，Apache 2.0）是 HTML→确定性 MP4 的本地渲染框架（CLI：init/lint/preview/render，Node 22+ + ffmpeg）。精剪 = 一个 hyperframes composition：粗剪成片作视频轨（data-track），字幕/图表/动效/B-roll 作叠加层。P3「不稳定」根因是每次生成的 HTML 不同，治理手段：(a) 固化 design.md 主题与组件模板，LLM 只填内容不写样式；(b) 渲染前必过 `hyperframes lint`；(c) 同输入同输出，回归可比对帧。

**FR-11b 精剪分镜规格 video-spec.md**：借鉴 feicaiclub/video-spec-builder 模板，精剪所有决策（哪个时间码出什么图表/B-roll/动效）先落成 video-spec.md，Chen 确认后才渲染。它是精剪的"EDL"：人类可读、逐行可改、改后重渲染。

**FR-12 图表与动态效果（skill: `broll-graphics`）**：从定稿提取数据点 → 生成图表提案（标题、数据、出现时间码）→ Chen 确认 → 渲染插入。

**FR-13 B-roll**：从定稿按段落生成 B-roll 需求清单（关键词、时长、插入点）→ 从素材库/授权来源匹配 → Chen 确认 → 装配。

**FR-14 风格对标**：以「小Lin说」为参考建立呈现 checklist（信息密度、图表出现频率、字幕节奏、B-roll 占比），作为精剪验收 rubric 的一部分。Sprint 0 由 agent 拆解 2–3 期参考视频形成量化 checklist。

## 5. EDL 规范（核心中间产物）

JSON + 同步生成 CSV（方便人看人改）：

```json
{
  "source": "raw_20260611.mp4",
  "fps": 30,
  "segments": [
    {
      "id": 17,
      "keep": true,
      "start": "00:03:12.480",
      "end": "00:03:18.920",
      "text": "所以美联储这次降息其实……",
      "script_line": 42,
      "pad_in_ms": 150,
      "pad_out_ms": 180,
      "note": ""
    },
    {
      "id": 18,
      "keep": false,
      "start": "00:03:18.920",
      "end": "00:03:24.100",
      "text": "所以美联储这个这个……（NG）",
      "reason": "duplicate_of:17, kept_last=false→改判保留17，理由：17更流畅",
      "decided_by": "llm"
    }
  ]
}
```

约定：`decided_by` ∈ {rule, llm, human}；人工改过的行标 human，重跑时不被覆盖。

## 6. 非功能需求

| ID | 要求 | 度量 |
|---|---|---|
| NFR-1 | 质量优先（2026-06-11 Chen 修订） | 不为省 token 牺牲识别精度与决策质量：ASR 用可用的最大模型；LLM 语义层对全部疑似重复和脱稿区间逐段复核。一次合格输出优于多次返工 |
| NFR-2 | 可复现性 | 同输入两次运行 EDL 完全一致（LLM 决策固定 temperature=0 + 缓存） |
| NFR-3 | 性能 | 20 分钟素材，粗剪全流程 ≤ 30 分钟（不含超分） |
| NFR-4 | 可微调 | 改 EDL 单行 → 重渲染 ≤ 5 分钟 |
| NFR-5 | 可观测 | 每步落盘中间产物，失败可从断点续跑 |

## 7. 验收标准（粗剪质量门槛）

用 Chen 提供的真实素材测试，由 Test Agent 出报告、Claude inspect、Chen 终审：

| 指标 | 门槛 |
|---|---|
| 重复句残留 | 0（对齐可发现的重复必须 100% 捕获） |
| 语气词残留 | ≤ 2 处/10 分钟成片 |
| >0.8s 停顿残留 | 0 |
| 误删（把该留的删了） | ≤ 1 处/期，且可通过 EDL 一行恢复 |
| 气口生硬感 | Chen 盲听 10 个切点，"听不出剪辑痕迹" ≥ 8 个 |
| 字幕规格 | 字体/字号/颜色/阴影逐项截图比对，100% 符合 FR-10 |
